**Pilier** : Reporting & Report Studio

💡 **IFS Cloud tip**

Crystal Report server misbehaving in IFS Cloud? One culprit catches teams out: a wrong **`LocalNetworkIpRange`** in `main_config.json`.

• Symptoms read like a reporting or permissions problem
• Root cause is **network/infra config**, not the report itself

Before deep-diving into Crystal, validate the environment config first. A surprising share of "reporting" incidents are really network/environment settings wearing a reporting costume. (And since Crystal is being retired, treat each incident as one more nudge to plan the Report Studio move.)
