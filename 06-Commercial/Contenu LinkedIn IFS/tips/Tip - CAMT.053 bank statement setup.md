💡 **IFS Cloud tip**

CAMT.053 bank-statement import looks intimidating, but it's four steps — and one of them causes most of the pain.

• Define the **message codes** from your bank's file ← this is where it breaks
• Define the **external payment parameters**
• Drop the file in the application server **IN folder**
• It loads into the app for reconciliation

Every bank labels transactions differently, so map every code before blaming the setup. (Same flow across Apps 9/10/Cloud.) When an import "half works," it's almost always unmapped codes, not the engine.
