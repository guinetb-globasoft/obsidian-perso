💡 **IFS Cloud tip**

An assistant always pre-selects a radio option and you want it to start blank? Don't fight it in the page — you're looking at the wrong layer.

The control is bound to an attribute whose default is defined **deeper**: at client-metadata or LU/API level. The UI is just reflecting it faithfully.

• Change or clear the default **where the attribute is defined**
• Test the LU/API default, not only the screen

The reusable lesson: in IFS, when a value "appears from nowhere," trace it to where it's really set. Knowing the layer saves hours of poking at the page.
