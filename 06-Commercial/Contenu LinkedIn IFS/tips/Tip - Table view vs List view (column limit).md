💡 **IFS Cloud tip**

"How many columns can I show on a screen?" — a recurring question in the IFS community. The useful answer isn't a number, it's a design rule:

• **Table view** → no hard column limit. Add as many fields as the role genuinely needs.
• **List view** → constrained by screen real estate, not a fixed cap.

So if a role lives in the data (planners, controllers, schedulers), build a **Table view**. Keep **List views** for scannable, few-column overviews. Choosing the right control up front saves a painful personalization rework later.
