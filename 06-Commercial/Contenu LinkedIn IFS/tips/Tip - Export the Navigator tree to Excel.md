**Pilier** : Permissions & Roles

💡 **IFS Cloud tip**

Designing permission sets or functional roles? Don't click through menus — start from the **entire Navigator tree in Excel**.

• A Quick Report over the navigator nodes (using LEVEL / indentation) renders the full hierarchy
• You get node + projection + entry type on every line
• Map roles against that one sheet, then build permission sets from it

Designing roles **top-down** from the real menu structure beats discovering missing access one support ticket at a time. Get the map first; the access model almost designs itself.
