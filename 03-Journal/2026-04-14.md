
## Odoo — Instance perso

L'instance Odoo **perso** (`edu-perso-bguinet.odoo.com` — "Benoit Guinet (Perso)") n'est **pas à gérer comme les autres instances** (Globasoft, Guinet Group). Ne pas y appliquer les mêmes routines de suivi comptable ou de gestion.



**Objectif instance perso** : gérer uniquement les comptes bancaires pour faire une analyse des dépenses personnelles.



## Odoo perso — Bug connu company_id

**ATTENTION** : L'instance perso (`edu-perso-bguinet.odoo.com`) tourne sur **Odoo 19**. Sur cette version, le champ `company_id` **n'existe plus** sur le modèle `account.account` (plan comptable). Il a été remplacé par `company_ids` (many2many).

Conséquence : **ne jamais passer `company_id` aux outils `get_plan_comptable`** ou `search_record` sur `account.account` pour l'instance perso. Le paramètre provoque une erreur `ValueError: Invalid field account.account.company_id`.

En revanche, `company_id` fonctionne toujours pour les journaux (`get_journaux`) et les écritures (`account.move`, `account.move.line`).
